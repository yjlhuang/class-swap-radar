# 洗課表 Prompt（Claude / GPT 通用）

把下面整段貼給 Claude 或 GPT，連同你們學校的**班級課表 PDF**一起丟進去。它會先跟你確認幾件事，再產出「調課雷達」能吃的 JSON。

---

你是我的課表資料助手。我會給你一份**班級課表** PDF（一頁一班），請幫我轉成一個 JSON 陣列，之後要餵給一個叫「調課雷達」的網頁工具。

## 輸出格式

整份是一個陣列，每一「格課」是一筆物件：

```json
{ "cls": "501", "day": 1, "period": 1, "subject": "英語文", "teacher": "林安" }
```

- `cls`：班級，字串。
- `day`：星期，1–5 對應 一～五。
- `period`：節次，數字 1–8。
- `subject`：科目名，照課表寫。
- `teacher`：老師姓名。
- `fixed`：**選填**，只有固定節次才加，值為 `true`（見下）。

只輸出 JSON，不要多餘說明文字。空堂／沒有課的格子**不要**產生任何一筆。

## 動手前，先問我這幾件事（很重要）

1. **哪些課名算「固定節次」**（要標 `fixed: true`）？常見是班週會、朝會、週會、社團活動。這些是全校綁死、不可對調的時段，雷達要自動排除。請把我回答的課名都標 `fixed: true`。
2. **有沒有「沒寫老師」的格子**？例如有些學校星期五第六、七節寫「社團」但沒有老師名字——那通常代表**這兩節是固定的社團／班週會時間**（每週活動不一定）。遇到這種沒有老師的格子，`teacher` 填 `"—"`、並標 `fixed: true`，不要漏掉也不要瞎編老師。
3. **是否要化名**？如果我要匿名分享，請你把所有老師名字換成穩定的化名（同一個真名固定對到同一個化名），並另外給我一張「真名 ↔ 化名」對照表，方便我自己看。

## 轉檔時請注意

- **一次只處理一個部**：只轉高中部的班級（通常 18 班），不要混進國中部。
- **代／兼等小字**：課表上「代」「兼」這類註記不影響格子本身，`subject`／`teacher` 照實際上課的人與科目填即可，不用另外造欄位。
- **連堂**：同一位老師、同一班、相鄰兩節同一科（例如第三、四節都是同班美術），照樣各產一筆，不用合併——雷達會自己判斷連堂。
- **同班多科**：同一位老師在同一班教兩種科目（例如導師教國文又教彈性），照實際各產各的筆數。
- 看不清楚或拿不準的格子，**先列出來問我**，不要猜。

## 最後請幫我做這兩個檢查

1. **撞課檢查**：同一位老師不應該在同一個 `day`+`period` 出現兩筆。若有，列出來。
2. **重格檢查**：同一個 `cls` 在同一個 `day`+`period` 不應有兩筆。若有，列出來。

檢查沒問題後，把最終 JSON 給我（可下載成 `schedule.json`）。我會拿去匯入「調課雷達」。
