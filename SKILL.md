# 把班級課表 PDF 洗成調課雷達 JSON

> **狀態：v0.1 草稿。** 這份 skill 是在只用示範假資料的情況下寫的。**發佈或大量使用前，請先拿一份真實的班級課表 PDF 跑一次**，把真 PDF 才會出現的狀況（各校固定節次寫法、沒寫老師的格子、代/兼註記、跨部老師、連堂）補進「處理規則」與「常見陷阱」兩節，再把本行提醒刪掉。

## 目標

輸入：一份**班級課表** PDF（一頁一班，格子含科目＋老師）。
輸出：一個 JSON 陣列，每筆是一格課，格式如下，可下載成 `schedule.json` 供「調課雷達」網頁匯入。

```json
{ "cls": "501", "day": 1, "period": 1, "subject": "英語文", "teacher": "林安" }
```

欄位：`cls`(班級字串)、`day`(1–5)、`period`(1–8)、`subject`、`teacher`、`fixed`(選填 true)。
空堂不產生任何一筆。

## 為什麼用班級課表而非教師課表

班級課表以班為單位，18 班天生只含高中部，不必判斷老師屬哪個部；教師課表會混進國中部的課。所以**一律要求班級課表**；若老師只有教師課表，說明差異並請他改提供班級課表。

## 流程

1. **讀 PDF、盤點**：確認是班級課表、共幾班、節次與星期軸怎麼排。若是掃描檔（非文字 PDF），先提醒可能需要人工校對。
2. **開工前，向老師確認三件事**（逐項問，不要跳過）：
   - 哪些課名算**固定節次**（要標 `fixed:true`）——通常班週會、朝會、週會、社團活動。
   - 有沒有**沒寫老師**的格子——常見是「社團／班週會」不寫老師，代表該節是全校固定活動時段。這種 `teacher` 填 `"—"`、標 `fixed:true`。
   - 是否要**化名**——要的話建立穩定的真名↔化名對照（同名固定同化名），另附對照表給老師。
3. **轉檔**：逐班、逐格產生筆數。遵守下面的處理規則。拿不準的格子先列出來問，不要猜。
4. **自我檢查**（用程式或逐筆核對）：
   - 撞課：同一 `teacher` 不應在同一 `day+period` 出現兩筆。
   - 重格：同一 `cls` 不應在同一 `day+period` 出現兩筆。
   - 覆蓋：班數、每班節數是否合理。
   列出任何異常給老師確認。
5. **交付**：輸出最終 JSON，說明如何匯入（「調課雷達」右上「換／匯入課表資料」→貼上或上傳）。

## 處理規則

- 一次只轉一個部（高中部，通常 18 班），不混國中部。
- 代／兼等小字不另立欄位，`subject`／`teacher` 照實際授課填。
- 連堂（同師同班相鄰同科）照樣各產一筆，不合併——工具自行判斷。
- 同班多科（同師在同班教兩科）照實際各產各筆。
- 只輸出 JSON，不夾雜說明（說明另外講）。

## 常見陷阱（待真資料補強）

- 沒寫老師的固定活動格 → `teacher:"—"` + `fixed:true`，勿漏勿造假。
- 跨部老師在班級課表裡只出現高中部那幾節，屬正常；不要為了補滿而編造。
- 各校節次時間、早自習列、午休位置不同，只需正確對到 `period` 1–8 即可。

## 完成後

可另外提供一段「給對方老師的調課訊息」範本，或引導老師直接用「調課雷達」內建的分享功能產生。
