# 調課雷達 · class-swap-radar

一個給高中老師用的**單檔調課小工具**。點自己課表的某一節，馬上看到「這班還有哪些老師、誰能跟我對調」，並可在期初一步步試調、產生給對方的訊息。

沒有後端、沒有登入、沒有資料庫——就一個 `index.html`。可以本機雙擊打開、丟 GitHub Pages、或用 Claude 的 artifact publish 分享。

> 版本 v0.9.0 · Built by YJL Huang with Claude, Gemini and GPT
> 目前為 DEMO（內建假資料、全化名），等真課表洗好即可換上。
> 緣起與致謝見文末。

## 它能做什麼

- **調課雷達**：點你想搬走的一節 → 依同班任課老師分組，列出誰能對調、哪一節可換、換不了的原因，並標示連堂。
- **此刻課堂**：選時間（預設現在）→ 看每班當下誰在上課；也能查某位老師此刻空堂或在哪班。
- **期初連動試調**：按「確認對調」即在記憶體中換位、即時更新課表，可以再以新課表繼續調第二、第三步；每步不同顏色標記，支援「復原上一步」與「重置課表」。
- **截圖分享**：試調後依對方老師分組，一鍵複製一段可傳 Line 的訊息（含調課路徑），提醒你切到對方課表截圖。
- **匯入自己的課表**：右上「換／匯入課表資料」，貼上或上傳 JSON 即可換一整套資料。

## 三種用法

1. **只想試玩**：直接打開 `index.html`，內建示範資料就能點。
2. **用在自己學校**：把你們的班級課表洗成 JSON（見下），用右上匯入貼進去；或把 JSON 直接取代 `index.html` 裡最上面那段 `window.SCHEDULE = [...]`，再 publish／上架。
3. **他校老師**：同一個 `index.html` 就是空殼，匯入你們自己的 JSON 即可，程式完全不用改。

## 資料格式

每一「格課」是一筆物件，整份是一個陣列。範例見 `schedule.sample.json`。

```json
[
  { "cls": "501", "day": 1, "period": 1, "subject": "英語文", "teacher": "林安" },
  { "cls": "501", "day": 5, "period": 6, "subject": "班週會", "teacher": "林安", "fixed": true }
]
```

| 欄位 | 說明 |
|---|---|
| `cls` | 班級，字串，如 `"501"` |
| `day` | 星期，1–5（一～五） |
| `period` | 節次，1–8 |
| `subject` | 科目名 |
| `teacher` | 老師（可用化名） |
| `fixed` | 選填。`true` = 全校固定節次（班週會／社團／朝週會），雷達自動排除、不可對調 |

**用「班級課表」當來源，不要用「教師課表」。** 班級課表以 18 班為單位，天生只含高中部、不必判斷老師屬哪個部；教師課表則會把國中部的課混進來。

## 洗課表資料

把班級課表 PDF 交給 Claude 或 GPT，配合 `wash-prompt.md` 的指示，就能產出上面格式的 JSON。重點規則（如「社團／班週會沒寫老師 = 該節為固定時段」）都寫在那份 prompt 裡。Claude 使用者也可用 `SKILL.md` 打包成 Skill。

## 檔案

| 檔案 | 用途 |
|---|---|
| `index.html` | 完整單檔前端（含所有功能＋匯入） |
| `schedule.sample.json` | 示範假資料，也是格式範本 |
| `wash-prompt.md` | 給 Claude／GPT 洗 PDF 的通用指示 |
| `SKILL.md` | Claude Skill 草稿（v0.1，建議先洗過一次真資料再定稿） |
| `HANDOFF.md` | 給接手開發者／LLM 的架構與決策說明 |

## 緣起與致謝

本專案的發想，始於與 Gemini 討論另一位老師做的「課表查詢系統」——奕鈞老師的開源專案 [timetable-demo](https://github.com/littleyi22/timetable-demo)，以及另一位將它改造成「AI 轉檔＋自行匯入」流程的老師的分享（《半桶水的AI筆記》：https://halfmemo.com/ai-notes/ai-timetable-lookup-tool/），讓我看到「把課表 PDF 交給 AI 轉成結構化資料、再用純前端在本機查詢、資料不外流」這條路行得通，也影響了本專案洗課表與匯入的作法。

從與Gemini定案規格和功能，和GPT確認路線拉出架構，最後與Claude Opus定案並做出初步demo，
本專案為獨立重寫、未使用兩位老師的程式碼，定位也不太相同：
兩位老師的工具解決的是**查詢**（查班級、查老師、查空堂），本專案的核心是**調課**——點一節課，算出同班哪位老師能跟你對調。

謝謝這些老師願意把東西開源分享，讓後面的人有肩膀可以站😊
等拿到課表跑一次實際狀況以後還會再更新，希望讓老師們可以調課更順利！
